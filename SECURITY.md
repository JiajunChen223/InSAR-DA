# Security Policy

InSAR-DA is a research codebase and public data artifact. It should not contain private credentials, unpublished source data, private exports, or local runtime artifacts.

## Reporting a Vulnerability

Please open a private security advisory on GitHub if available, or contact the repository owner through the GitHub profile. Include:

- A concise description of the issue.
- Steps to reproduce it.
- The affected commit or release.
- Any relevant logs or stack traces with sensitive paths removed.

## Supported Versions

Security fixes are handled for the latest public release.
